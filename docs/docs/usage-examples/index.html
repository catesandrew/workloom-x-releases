Usage Examples

Validate a Workspace​

pnpm wl validate fixtures/workspaces/simple-workspace.json

Use this before rendering or syncing. Validation checks manifest shape and the current fixture trust gate.

Render Without Writing​

pnpm wl render --workspace fixtures/workspaces/simple-workspace.json --dry-run

This prints a deterministic render plan with file paths, hashes, providers, package names, and projection policies.

Check CI Drift​

pnpm wl sync --check --workspace fixtures/workspaces/simple-workspace.json

Clean workspaces exit 0. Drift and trust failures exit non-zero:

pnpm wl sync --check --workspace fixtures/workspaces/drift-provider-mismatch.json
pnpm wl sync --check --workspace fixtures/workspaces/policy-denied-hooks.json

Apply Approved Output​

pnpm wl sync --workspace fixtures/workspaces/committed-copilot-projection.json --approve-local

--approve-local is the current prototype approval mechanism. It is not an enterprise approval workflow and does not authorize hooks, MCP servers, or permission expansion.

Restore Sandbox Output​

pnpm wl restore --workspace fixtures/workspaces/simple-workspace.json

Current limitation: restore rewrites planned files from the render plan. Full ownership-safe restore, where only known Workloom-owned files can be restored, is still roadmap work.

Publish a Draft Package Manifest​

pnpm wl push --draft --package fixtures/packages/workloom-baseline-v1.json

The command hashes selected package assets. It does not upload arbitrary workspace folders.
